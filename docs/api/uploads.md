# Strava API v3 – Uploads

> Auto-generated by `scripts/scrape_strava_docs.py` from https://developers.strava.com/docs/reference

**Total endpoints:** 2

Uploading activities to Strava is an asynchronous operation — you POST the file, then poll `GET /uploads/{uploadId}` until the upload status becomes `"Your activity is ready."` or an error occurs.

---

## `POST` /uploads

**Operation ID:** `createUpload`  
**Summary:** Upload Activity  
**Required Scopes:** `activity:write`  

Uploads a new data file to create an activity. Requires a `multipart/form-data` request.

### Supported File Formats

| `data_type` value | Format |
|-------------------|--------|
| `fit` | FIT binary (Garmin, Wahoo, etc.) |
| `fit.gz` | gzip-compressed FIT |
| `tcx` | Training Center XML |
| `tcx.gz` | gzip-compressed TCX |
| `gpx` | GPS Exchange Format |
| `gpx.gz` | gzip-compressed GPX |

### Parameters

| Name | In | Required | Type | Description |
|------|----|----------|------|-------------|
| `file` | body | ✅ | `file` | The activity file to upload |
| `data_type` | body | ✅ | `string` | File format: `fit`, `fit.gz`, `tcx`, `tcx.gz`, `gpx`, `gpx.gz` |
| `name` | body | ❌ | `string` | Activity name |
| `description` | body | ❌ | `string` | Activity description |
| `trainer` | body | ❌ | `string` | `"1"` if recorded on a trainer |
| `commute` | body | ❌ | `string` | `"1"` if this is a commute |
| `external_id` | body | ❌ | `string` | External ID (typically the source filename) |

### Response

Returns: **`Upload`** (initial status — may be `"processing"`)

```json
{
  "id": 9876543210,
  "id_str": "9876543210",
  "external_id": "my_activity.fit",
  "error": null,
  "status": "Your activity is ready.",
  "activity_id": 1234567890
}
```

### Upload Status Values

| Status | Description |
|--------|-------------|
| `"Waiting..."` | Queued for processing |
| `"Processing..."` | Currently processing |
| `"Your activity is ready."` | Successfully created |
| `"There was an error processing your activity."` | Processing failed |

### Example

```bash
$ http --form post "https://www.strava.com/api/v3/uploads" \
  "Authorization: Bearer [[token]]" \
  file@"recording.fit" \
  data_type="fit" \
  name="Morning Ride" \
  external_id="recording.fit"
```

## `GET` /uploads/{uploadId}

**Operation ID:** `getUploadById`  
**Summary:** Get Upload  
**Required Scopes:** `activity:write`  

Returns an upload for a given identifier. Poll this endpoint after creating an upload until `activity_id` is populated or `error` is not null.

### Parameters

| Name | In | Required | Type | Description |
|------|----|----------|------|-------------|
| `uploadId` | path | ✅ | `long` | The identifier of the upload |

### Response

Returns: **`Upload`**

| Field | Type | Description |
|-------|------|-------------|
| `id` | `long` | Upload identifier |
| `id_str` | `string` | Upload identifier as string |
| `external_id` | `string` | External identifier supplied on creation |
| `error` | `string` | Error message if processing failed (null otherwise) |
| `status` | `string` | Upload processing status |
| `activity_id` | `long` | Created activity ID (null while processing) |

### Polling Pattern

```python
import time
import httpx

upload_id = create_upload_response["id"]

while True:
    status = client.get(f"/uploads/{upload_id}").json()
    if status["activity_id"] is not None:
        print(f"Activity created: {status['activity_id']}")
        break
    if status["error"]:
        raise RuntimeError(f"Upload failed: {status['error']}")
    time.sleep(2)  # Poll every 2 seconds
```

### Example

```bash
$ http get "https://www.strava.com/api/v3/uploads/{uploadId}" \
  "Authorization: Bearer [[token]]"
```

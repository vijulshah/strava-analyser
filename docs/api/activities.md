## `POST` /activities

**Operation ID:** `createActivity`  
**Summary:** Create a Manual Activity  
**Required Scopes:** `activity:write`  

Creates a manual activity for the authenticated athlete.

### Parameters

| Name | In | Required | Type | Description |
|------|----|----------|------|-------------|
| `name` | body | ✅ | `string` | Activity name |
| `sport_type` | body | ✅ | `string` | Sport type (e.g. Run, Ride, Swim) |
| `start_date_local` | body | ✅ | `datetime` | ISO 8601 local start time |
| `elapsed_time` | body | ✅ | `integer` | Elapsed time in seconds |
| `description` | body | ❌ | `string` | Activity description |
| `distance` | body | ❌ | `float` | Distance in meters |
| `trainer` | body | ❌ | `integer` | 1 if trainer/indoor activity |
| `commute` | body | ❌ | `integer` | 1 if commute |

### Response

Returns: **`DetailedActivity`**

### Example

```bash
$ http post "https://www.strava.com/api/v3/activities" \
  "Authorization: Bearer [[token]]" \
  name="Morning Run" sport_type="Run" \
  start_date_local="2024-01-15T07:30:00Z" \
  elapsed_time:=3600 distance:=10000
```

## `GET` /athlete/activities

**Operation ID:** `getLoggedInAthleteActivities`  
**Summary:** List Athlete Activities  
**Required Scopes:** `activity:read`  

Returns the activities of an athlete for a specific identifier. Requires `activity:read`. Only Me activities will be filtered out unless requested by a token with `activity:read_all`.

> **Key filtering endpoint** — supports epoch timestamp-based date range filtering.

### Parameters

| Name | In | Required | Type | Description |
|------|----|----------|------|-------------|
| `before` | query | ❌ | `integer` | Epoch Unix timestamp — return activities *before* this time |
| `after` | query | ❌ | `integer` | Epoch Unix timestamp — return activities *after* this time |
| `page` | query | ❌ | `integer` | Page number (default 1) |
| `per_page` | query | ❌ | `integer` | Number of items per page (default 30, max 200) |

### Time Filtering Examples

| Filter | Description | before | after |
|--------|-------------|--------|-------|
| Today | Activities from midnight today | `now()` | `today_midnight()` |
| Last 7 days | Past week | `now()` | `now() - 7 days` |
| Last 14 days | Past 2 weeks | `now()` | `now() - 14 days` |
| Last 30 days | Past month | `now()` | `now() - 30 days` |
| Last 3 months | Past quarter | `now()` | `now() - 90 days` |
| Last 6 months | Past 6 months | `now()` | `now() - 180 days` |
| Last year | Past 365 days | `now()` | `now() - 365 days` |
| This year | Jan 1 to now | `now()` | `Jan 1 00:00:00` |
| Custom | Any range | `custom_end` | `custom_start` |

### Response

Returns: **`SummaryActivity[]`**

### Example

```bash
$ http get "https://www.strava.com/api/v3/athlete/activities?before=&after=&page=1&per_page=30" \
  "Authorization: Bearer [[token]]"
```

## `GET` /activities/{id}/zones

**Operation ID:** `getZonesByActivityId`  
**Summary:** Get Activity Zones  
**Required Scopes:** `activity:read`  

Returns the heart rate and power zones of the given activity. Requires a premium athlete; Requires `activity:read_all` for Only Me activities.

### Parameters

| Name | In | Required | Type | Description |
|------|----|----------|------|-------------|
| `id` | path | ✅ | `long` | The identifier of the activity |

### Response

Returns: **`ActivityZone[]`**

Each zone object includes:
- `score` — overall zone score
- `distribution_buckets` — `TimedZoneDistribution` array (time in each zone)
- `type` — `"heartrate"` or `"power"`
- `sensor_based` — whether data comes from a sensor
- `custom_zones` — whether custom zones were used
- `max` — max HR or power value

### Example

```bash
$ http get "https://www.strava.com/api/v3/activities/{id}/zones" \
  "Authorization: Bearer [[token]]"
```
